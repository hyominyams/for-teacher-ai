# For Teacher AI - Project Rules (v3.0)

## 1. 프로젝트 개요 (Project Overview)
- **제품명**: For Teacher AI
- **목적**: 교사의 생활기록부 작성 효율을 극대화하는 AI 기반 작업 플랫폼.
- **현재 진행도**: 행동특성 및 창체활동 워크스페이스 완료. 교과세특 개발 단계 진입.

## 2. 인터페이스 및 UX 규칙 (UI/UX Rules)
- **통합 대시보드 (`/app`)**:
  - 각 영역(행동, 창체, 교과)을 탭으로 전환하는 통합 워크스페이스 구조.
  - 프리미엄 다크/라이트 테마 및 글래스모피즘 UI 적용.
  - 내비게이션: `NavbarMain`을 통해 대시보드와 작업로그 간 빠른 이동 지원.
- **작업 공간 (Workspaces)**:
  - **3단 구성**: 상단(글자수 설정/업로드), 좌측(기능 탭), 우측(학생별 작업 카드 그리드).
  - **반응형**: 모바일 메뉴 제공 및 데스크탑 최적화 그리드 배치.

## 3. 기능 구현 및 데이터 규칙
- **데이터 저장 (`work_logs` 테이블)**:
  - 사용자별(`user_id`), 카테고리별(`category`) 단일 레코드를 유지하며 `jsonb` 필드(`data`)에 전체 학생 상태 저장.
  - **Auto-save**: 입력 값 변경 시 3초 디바운스 로직을 통한 자동 저장 수행.
- **파일 연동**:
  - CSV 업로드: '번호'를 기준으로 기존 데이터 매핑 또는 신규 학생 생성.
  - CSV 내보내기: 미리보기 모달 및 작업로그 페이지에서 영역별 데이터 추출 지원.
- **보안**: 
  - Supabase Auth를 통한 세션 관리.
  - 학생 실명 저장 지양 (번호 기반 식별).

## 4. 관련 규칙 문서
- `behavior-workspace-rules.md`: 행동특성 워크스페이스 세부 규칙
- `creative-workspace-rules.md`: 창체활동 워크스페이스 세부 규칙
- `subject-workspace-rules.md`: 차기 교과세특 개발 가이드
- `auth-rules.md`: 인증 처리 규칙
